# Screenshot Gallery

This directory contains curated screenshots for the Ashmii Website Recovery Case Study.

The screenshots are organized into three stages:

- `before/` – Initial condition and visible recovery issues.
- `recovery/` – Infrastructure, database, plugin, media, security, and validation work.
- `after/` – Final restored website views and post-recovery validation.

## Folder Guide

| Folder | Purpose |
|---|---|
| `before/` | Documents the initial broken or incomplete recovery state. |
| `recovery/cloudflare/` | Cloudflare and DNS recovery evidence. |
| `recovery/database/` | Database import, reconstruction, and WordPress database upgrade screens. |
| `recovery/plugins/` | Theme, WPBakery, Revolution Slider, and required plugin recovery. |
| `recovery/media/` | Wayback Machine and media reconstruction evidence. |
| `recovery/security/` | Malware and spam URL injection investigation screenshots. |
| `recovery/validation/` | Broken image/link validation and final QA evidence. |
| `after/` | Restored desktop/mobile website and final functional views. |

## Publishing Note

Before pushing these screenshots to a public repository, manually review each image and redact any sensitive information such as emails, account IDs, IP addresses, license keys, server paths, or private project data.
